Introduction

This document is an application note that details to use of hardware multicast filtering on the Broadcom line of internal Ethernet MACs used in Broadcoms digital video line of chipsets.

There are two prerequisites to using multicast under Linux on systems that use one of the preceding chips. First, the Linux kernel must be configured to handle multicast. This can be selected during kernel compile time as a kernel configuration option. Instructions for doing this are given in the next section.

Second, the driver is programmed to make use of the 8 perfect match registers in the chip.  These registers can be programmed to receive specific destination MAC addressed packets. As this filtering is done in hardware, the packet handling is very efficient. Of these registers, one is always used for the MAC address of the system; the others are available for hardware multicast filtering.

The technique used to implement hardware filtering in the Broadcom driver is based on code examples provided in Linux Device Drivers, 2nd Edition, by Allessandro Rubini and Jonathan Corbet, pp 462-464.

Building The Kernel with Multicast Support

In order to support multicast, the Linux kernel must be configured with IP Multicasting. The Linux kernel is capable of software filtering of IP multicast data if the Ethernet device driver has the ability to pass all multicast traffic to the IP stack in the kernel. Depending on the amount of multicast traffic on the network, this software filtering can significantly degrade the system performance. Hardware filtering should improve system performance in cases where the following are true:

1.       A Large amount of multicast traffic is present on the network.

2.       Each client is registered for a small number of multicast groups (less than the maximum number of perfect match registers).

The STB Linux kernels are configured to support multicast (CONFIG_IP_MULTICAST=y) by default.

 

Multicast Usage

The appendices include two simple Linux programs for testing multicast. The first program allows the user to broadcast characters or character strings over multicast sockets. An example of executing this program is:

 

./multisender 224.0.0.10 1234 this is a test

 

The second program is a multicast receiver application. In can receive the messages from the first program. An example of executing this program is:

 

./multireceiver 224.0.0.10 1234

 

The receiver program should be started before the sender so that it is ready to receive the data from the sender.

Both programs are self contained and can be built with gcc or a cross compiler version of gcc. An example compile command is:

 

mipsel-linux-gcc o multisender_le multisender.c

 

Many small numbered sockets are protected and will cause bind() to return an error if the user is not the super user. It is best to use sockets above 1000. Programs that terminate with errors can be quickly debugged by using strace to trace system calls into the kernel.

 

Questions and Answers

Q. Is it normal for the driver to register for address 224.0.0.1?

A. Yes, this is the group address for IGMP messages.

 

Q. What are the valid addresses for multicast?

A. As per RFC 1112, page 3, host group addresses are identified by class D addresses. This range would be 224.0.0.0 to 239.255.255.255.

 

Q. Should 224.0.0.1 translate to MAC address 01:00:5e:00:00:01?

A. Yes, as per RFC 1112, page 3, host group addresses are identified by class D addresses, ie, those with 1110 as their high order bits. OReillys online Networking as a Second Language ,2nd Edition, http://www.oreillynet.com/pub/a/network/2001/08/10/net_2nd_lang.html, confirms that the MAC address range is 0x0100.5Enn.nnnn.

 

Known Limitations

1.       Because the Broadcom internal MAC has a limited number of perfect match registers, the maximum number of multicast addresses registered at any given time should be less than this number. Testing has also shown that IGMP will use one of these addresses as well as the users MAC address. If all perfect match registers are in use, the driver reverts back to passing all multicast packets to the IP stack and letting the kernel filter the packets in software. If the usage falls below the register limit, the driver reinstates the hardware filtering scheme.

#include <sys/types.h>

#include <sys/socket.h>

#include <netinet/in.h>

#include <arpa/inet.h>

#include <netdb.h>

#include <stdio.h>

#include <unistd.h> /* close */

 

#define MAX_MSG 100

 

int main(int argc, char *argv[]) {

 

int sd, rc, i;

unsigned char ttl = 1;

struct sockaddr_in cliAddr, servAddr;

struct hostent *h;

 

if(argc<4) {

printf("usage %s <mgroup> <port> <data1> <data2> ... <dataN>\n",argv[0]);

exit(1);

}

 

h = gethostbyname(argv[1]);

if(h==NULL) {

printf("%s : unknown host '%s'\n",argv[0],argv[1]);

exit(1);

}

 

servAddr.sin_family = h->h_addrtype;

memcpy((char *) &servAddr.sin_addr.s_addr, h->h_addr_list[0],h->h_length);

servAddr.sin_port = htons(atoi(argv[2]));

/* check if dest address is multicast */

if(!IN_MULTICAST(ntohl(servAddr.sin_addr.s_addr))) {

printf("%s : given address '%s' is not multicast \n",argv[0],

inet_ntoa(servAddr.sin_addr));

exit(1);

}

 

/* create socket */

sd = socket(AF_INET,SOCK_DGRAM,0);

if (sd<0) {

printf("%s : cannot open socket\n",argv[0]);

exit(1);

}

/* bind any port number */

cliAddr.sin_family = AF_INET;

cliAddr.sin_addr.s_addr = htonl(INADDR_ANY);

cliAddr.sin_port = htons(0);

if(bind(sd,(struct sockaddr *) &cliAddr,sizeof(cliAddr))<0) {

perror("bind");

exit(1);

}

 

if(setsockopt(sd,IPPROTO_IP,IP_MULTICAST_TTL, &ttl,sizeof(ttl))<0) {

printf("%s : cannot set ttl = %d \n",argv[0],ttl);

exit(1);

}

 

printf("%s : sending data on multicast group '%s' (%s)\n",argv[0],

h->h_name,inet_ntoa(*(struct in_addr *) h->h_addr_list[0]));

 

/* send data */

for(i=3;i<argc;i++) {

rc = sendto(sd,argv[i],strlen(argv[i])+1,0,

(struct sockaddr *) &servAddr, sizeof(servAddr));

if (rc<0) {

printf("%s : cannot send data %d\n",argv[0],i-1);

close(sd);

exit(1);

}

 

}/* end for */

 

/* close socket and exit */

close(sd);

exit(0);

}

 

 

Appendix B Multicast Receiver Test Program

/* fpont 12/99 */

/* pont.net */

/* mcastServer.c */

 

#include <sys/types.h>

#include <sys/socket.h>

#include <netinet/in.h>

#include <arpa/inet.h>

#include <netdb.h>

#include <stdio.h>

#include <unistd.h> /* close */

 

#define MAX_MSG 100

 

int main(int argc, char *argv[]) {

 

int sd, rc, n, cliLen;

struct ip_mreq mreq;

struct sockaddr_in cliAddr, servAddr;

struct in_addr mcastAddr;

struct hostent *h;

char msg[MAX_MSG];

 

if(argc!=3) {

printf("usage : %s <mcast address> <port>\n",argv[0]);

exit(0);

}

 

/* get mcast address to listen to */

h=gethostbyname(argv[1]);

if(h==NULL) {

printf("%s : unknown group '%s'\n",argv[0],argv[1]);

exit(1);

}

memcpy(&mcastAddr, h->h_addr_list[0],h->h_length);

/* check given address is multicast */

if(!IN_MULTICAST(ntohl(mcastAddr.s_addr))) {

printf("%s : given address '%s' is not multicast\n",argv[0],

inet_ntoa(mcastAddr));

exit(1);

}

 

/* create socket */

sd = socket(AF_INET,SOCK_DGRAM,0);

if(sd<0) {

printf("%s : cannot create socket\n",argv[0]);

exit(1);

}

 

/* bind port */

servAddr.sin_family=AF_INET;

servAddr.sin_addr.s_addr=htonl(INADDR_ANY);

servAddr.sin_port=htons(atoi(argv[2]));

if(bind(sd,(struct sockaddr *) &servAddr, sizeof(servAddr))<0) {

printf("%s : cannot bind port %d \n",argv[0],argv[2]);

exit(1);

}

 

/* join multicast group */

mreq.imr_multiaddr.s_addr=mcastAddr.s_addr;

mreq.imr_interface.s_addr=htonl(INADDR_ANY);

rc = setsockopt(sd,IPPROTO_IP,IP_ADD_MEMBERSHIP,

(void *) &mreq, sizeof(mreq));

if(rc<0) {

printf("%s : cannot join multicast group '%s'",argv[0],

inet_ntoa(mcastAddr));

exit(1);

}

else {

printf("%s : listening to mgroup %s:%s\n",

argv[0],inet_ntoa(mcastAddr), argv[2]);

 

/* infinite server loop */

while(1) {

cliLen=sizeof(cliAddr);

n = recvfrom(sd,msg,MAX_MSG,0,(struct sockaddr *) &cliAddr,&cliLen);

if(n<n) {

printf("%s : cannot receive data\n",argv[0]);

continue;

}

 

printf("%s : from %s:%d on %s : %s\n",argv[0],

inet_ntoa(cliAddr.sin_addr),ntohs(cliAddr.sin_port),

argv[1],

msg);

}/* end of infinite server loop */

 

}

 

return 0;

 

}
